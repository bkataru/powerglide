      138  
       46 -At the heart of powerglide is the **Ralph Loop** — a state mac
          -hine that drives agent behavior:                              
      139 +At the heart of powerglide is the **Ralph Loop** — a state mac
          +hine inspired by [ralph](https://github.com/snarktank/ralph) t
          +hat drives every agent session:                               
      140  
      141  ```
       49 -idle → thinking → acting → observing → (repeat or finish)     
      142 +IDLE                                                          
      143 +  │                                                           
      144 +  ▼                                                           
      145 +LOAD_TASKS ──(all done?)──► COMPLETE                          
      146 +  │                                                           
      147 +  ▼                                                           
      148 +PICK_TASK (highest priority, passes == false)                 
      149 +  │                                                           
      150 +  ▼                                                           
      151 +THINKING ──(tool_call)──► ACTING ──(result)──► OBSERVING      
      152 +  │                                               │           
      153 +  ▼                                               │           
      154 +VERIFY (run checks)                               │           
      155 +  │                                               │           
      156 +  ▼                                               │           
      157 +COMMIT ──► update task status ──► back to LOAD_TASKS          
      158 +  │                                                           
      159 +  ▼                                                           
      160 +FAILED (error recovery / retry)                               
      161  ```
      162  
       52 -Each iteration represents one thinking-acting-observing cycle,
          - with configurable velocity (delay between iterations).       
      163 +The loop emits `<POWERGLIDE_DONE>` when all tasks are complete
          +, which the orchestrator detects as a clean stop signal.

      165 +### Velocity Control                                          
      166

      168 +

          +eps

      173 +                                                              
      174 +Agents can self-adjust velocity by writing `VELOCITY=<value>`

      175 +                                                              
      176 +### Rogue Agent Prevention                                    
      177 +                                                              
      178 +powerglide includes multi-layer rogue agent prevention:       
      179 +                                                              
      180 +| Mechanism | Default | Description |                         
      181 +|-----------|---------|-------------|                         
      182 +| Step limit | 200 | Kill after N steps regardless |          
      183 +| Heartbeat | 30s | Worker must write heartbeat; monitor kills

          +same args 3+ times |

          +xceeded |

          +E_DONE` to terminate cleanly |
