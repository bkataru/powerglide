Architecture

powerglide is built in Zig 0.15.2 with a modular architecture designed for reliability and performance.

Module Structure

The Ralph Loop

At the heart of powerglide is the Ralph Loop — a state machine that drives every agent session:

pub const LoopState = enum {
    idle,
    thinking,
    acting,
    observing,
    finished,
    failed,
};

State Transitions

The loop emits <POWERGLIDE_DONE> when all tasks are complete.

Multi-Agent Swarm Architecture

Inter-agent communication is file-based at ~/.powerglide/teams/{team-id}/messages/.

Rogue Agent Prevention

powerglide includes multi-layer rogue agent prevention:

MechanismDefaultDescription
Step limit200Kill after N steps regardless
Heartbeat30sWorker must write heartbeat; monitor kills if missed
Circuit breaker3 repeatsKill if same tool called with same args 3+ times
Budget trackingconfigurableStop if token/cost budget exceeded
Explicit done signalrequiredAgent must emit POWERGLIDE_DONE to terminate cleanly

Velocity Control

Velocity controls how fast the ralph loop progresses:

powerglide run --velocity 2.0  # 2x speed: 500ms between steps
powerglide run --velocity 0.5  # 0.5x speed: 2000ms between steps

Agents can self-adjust velocity by writing VELOCITY=<value> to their session config file at ~/.config/powerglide/session-<id>.json. The orchestrator polls this file every N steps.

Configuration Loading

Configuration is loaded in order of precedence:

  1. Command-line arguments (highest priority)
  2. Environment variables
  3. ~/.config/powerglide/config.json
  4. Built-in defaults (lowest priority)
const config = Config.load(.{
    .cli_args = args,
    .env = std.process.env,
    .config_file = "~/.config/powerglide/config.json",
});

        // Implementation
    }

pub const ToolResult = struct {
    success: bool,
    output: []const u8,
